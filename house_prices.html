3.1. Model selection¶

In order to validate each of the models, 3-fold crossvalidation is employed. The mentioned NIA is used as random state in order to have reproducible results. Mention that the average score (the RMSE error in this case) for the 3-fold crossvalidation is stored as well as the spent time in two dictionaries. Only the training data should be used in this part, in order to avoid obtaining too optimistic results when models are going to be evaluated with the test set.

In [5]:
from sklearn.model_selection import cross_val_score, KFold

cv = KFold(n_splits=3, shuffle=True, random_state=NIA)

error_dic = {} # used to store the mean error for each model validation
time_dic = {}  # used to store the spent time of each model

Next function prints the mean RMSE and spent time in seconds for selected models.

In [6]:
def show_stats(keys = 'all'):
    '''Print mean RMSE and spent time in seconds
    
    args:
        key (str/list): keys to access to both dictionaries (error and test)
        
    '''
    print('-'*61)
    print(f"| {'MODEL':<27} | {'RMSE':<12} | {'TIME(SEC)':<12} |")
    print('-'*61)
    if keys == 'all':
        keys = list(error_dic.keys())
    elif isinstance(keys, str):
        keys = [keys]
    for key in keys:
        print(f"| {key:<27} | {round(error_dic[key], 5):<12} | {round(time_dic[key], 5):<12} |")
        print('-'*61)

params_dic = {} # used to store the selected hyper-parameters 

def show_best_params(key):
    '''Print selected hyper-parameters of a model
    
    args:
        key (str): key to access to the model
        
    '''
    print('-'*61)
    print(f"| {f'Params {key}':<34} | {'Value':<20} |")
    print('-'*61)
        
    for param, value in params_dic[key].items():
        if param == 'scaler':
            continue
        if isinstance(value, float):
            value = round(value, 5)
        print(f"| {param:<34} | {value:<20} |")
        print('-'*61)

3.1.2. Hyper-parameter tuning of KNN using grid-search¶

In this section is explored the KNN model. It is only evaluated its main parameter, the number of neighbors, with a grid-search. In this configuration a model is trained per each combination of parameters and the best one respect the scoring (in this case the RMSE) is selected.

In order to find the best number of neighbors, diferent ranges have been explored until it has been found the error was not decreasing.

3.1.3. Hyper-parameter tuning of Decision Tree using random-search¶

In this case random-search is used to tune the Decision Tree, which tests randomly only some of the combinations. The considered hyper-parameters are:

  • Maximum depth: it is going to determine the complexity of the model. A higher degree of complexity allows to explain better the problem, but it can lead to overfitting.
  • Minimum number of samples required to split and internal node: according to the paper, An empirical study on hyperparameter tuning of decision trees (https://arxiv.org/abs/1812.02207, recommended values are between 1 and 40.
  • Minimumn number of samples required to be at a leaf node: according to the previous mentioned paper, the recommended values are between 1 and 20.

3.1.5. Hyper-parameter of Histogram Gradient Boosting using optuna¶

For the last model it is employed optuna. The explored hyperparameters are:

  • Learning rate: also known as shrinkage. This is used as a multiplicative factor for the leaves values.
  • Maximum number of iterations: the maximum number of iterations of the boosting process (maximum number of trees).
  • Maximum number of leaves for each tree.

In optuna it is required to call an objective function where it is calculated the score.

In [18]:
import optuna
from optuna.samplers import TPESampler

def objective(trial):
    
    # Search space
    learning_rate = trial.suggest_float('learning_rate', 0.01, 0.9)    # learning rate
    max_iter = trial.suggest_int('max_iter', 50, 150)                  # maximum number of iterations
    max_leaf_nodes = trial.suggest_int('max_leaf_nodes', 10, 40)       # maximum number of leaves for each tree
    
    # Model
    clf = HistGradientBoostingRegressor(learning_rate=learning_rate, 
                                        max_iter=max_iter,
                                        max_leaf_nodes=max_leaf_nodes,
                                        random_state = NIA)
    # Training
    scores = -cross_val_score(clf, X_train, y_train, 
                              scoring='neg_root_mean_squared_error', 
                              n_jobs=-1,
                              cv=cv)
    return scores.mean()

Once the objective function is declared, it is defined a number of trials (in this case 15) and it is defined if the objective should be minimized of maximized (in this case minimize the RMSE). The selected hyperparameters of the trial with the best result are selected.

Pruning Unpromising Trials¶

This feature automatically stops unpromising trials at the early stages of the training (automated early-stopping). For the case of plotting intermediate values it is needed a regression model capable of doing partial fits as SDG. This is why it is used this regressor for the tuning of parameters.

In [25]:
from optuna.visualization import plot_intermediate_values
from sklearn.linear_model import SGDRegressor

X_train_2, X_val, y_train_2, y_val = train_test_split(X_train, y_train, test_size=1/3, random_state=NIA)

def objective_2(trial):
    
    # Search space
    l1_ratio = trial.suggest_float('l1_ratio=0.15', 0.1, 0.3)    # learning rate
    max_iter = trial.suggest_int('max_iter', 500, 1500)          # maximum number of leaves for each tree
    
    # Model
    clf = SGDRegressor(l1_ratio=l1_ratio, 
                       max_iter=max_iter,
                       random_state=NIA)
    
    for step in range(100):
        clf.partial_fit(X_train_2, y_train_2)

        # Report intermediate objective value.
        intermediate_value = -clf.score(X_val, y_val)
        trial.report(intermediate_value, step)

        # Handle pruning based on the intermediate value.
        if trial.should_prune():
            raise optuna.exceptions.TrialPruned()

    return clf.score(X_val, y_val)

In [31]:
def objective(trial):
    
    # Search space for model
    method = trial.suggest_categorical('method', ['ExtraTrees', 'HistGradientBoosting'])
    
    # ExtraTrees
    if(method == 'ExtraTrees'):
        
        # Search space hyper-parameters
        max_depth = trial.suggest_int("max_depth", 1, 16)                   # maximum depth of the tree
        min_samples_split = trial.suggest_int("min_samples_split", 2, 40)   # minimum number of samples required to split an internal node
        min_samples_leaf = trial.suggest_int("min_samples_leaf", 1, 20)     # minimum number of samples required to be at a leaf node
        
        # Model
        clf = ExtraTreesRegressor(max_depth=max_depth, 
                                  min_samples_split=min_samples_split,
                                  min_samples_leaf=min_samples_leaf,
                                  random_state = NIA)
        
    # HistGradientBoosting
    else:
        
        # Search space hyper-parameters
        learning_rate = trial.suggest_float('learning_rate', 0.01, 0.9)      # learning rate
        max_iter = trial.suggest_int('max_iter', 50, 150)                    # maximum number of iterations
        max_leaf_nodes = trial.suggest_int('max_leaf_nodes', 10, 40)         # maximum number of leaves for each tree
        
        # Model
        clf = HistGradientBoostingRegressor(learning_rate=learning_rate, 
                                            max_iter=max_iter,
                                            max_leaf_nodes=max_leaf_nodes,
                                            random_state = NIA)
        
    # Training
    scores = -cross_val_score(clf, X_train, y_train, 
                              scoring='neg_root_mean_squared_error', 
                              n_jobs=1, cv=cv)
    return scores.mean()

3.4.1. Halving hyper-parameter optimization¶

Halving Grid Search CV¶

This optimization method consist in evaluating all the possible combinations of hyper-parameters, but in a small subset of the training data. The candidates which obtain better results are selected and the others are discarded. Then, this reduced group of candidates are trained with more data to select the new best candidates. The process is repeated until all data is used.

lightgbm¶

Lightgbm works in a very similar way as XGBoost does but the main difference here is in the construction of trees. LightGBM does not grow a tree level-wise row by row as most other implementations do. Instead it grows trees leaf-wise. It chooses the leaf it believes will yield the largest decrease in loss. Another characteristic of lightGBM is that it utilizes two novel techniques called Gradient-Based One-Side Sampling (GOSS) and Exclusive Feature Bundling (EFB) which allow the algorithm to run faster while maintaining a high level of accuracy

model_cb.fit(X, y)

# Competition prediction
y_pred = model_cb.predict(X_comp)

# Exponential transformation for competition
salePrice = np.exp(y_pred)

submission_cb = pd.DataFrame(data={'Id': i, 'SalePrice': salePrice})

submission_cb.to_csv('submission_catboost.csv', index=False)

Examples
--------

    The simplest way to use nbconvert is

            > jupyter nbconvert mynotebook.ipynb --to html

            Options include ['asciidoc', 'custom', 'html', 'latex', 'markdown', 'notebook', 'pdf', 'python', 'rst', 'script', 'slides', 'webpdf'].

            > jupyter nbconvert --to latex mynotebook.ipynb

            Both HTML and LaTeX support multiple output templates. LaTeX includes
            'base', 'article' and 'report'.  HTML includes 'basic', 'lab' and 
            'classic'. You can specify the flavor of the format used.

            > jupyter nbconvert --to html --template lab mynotebook.ipynb

            You can also pipe the output to stdout, rather than a file

            > jupyter nbconvert mynotebook.ipynb --stdout

            PDF is generated via latex

            > jupyter nbconvert mynotebook.ipynb --to pdf

            You can get (and serve) a Reveal.js-powered slideshow

            > jupyter nbconvert myslides.ipynb --to slides --post serve

            Multiple notebooks can be given at the command line in a couple of 
            different ways:

            > jupyter nbconvert notebook*.ipynb
            > jupyter nbconvert notebook1.ipynb notebook2.ipynb

            or you can specify the notebooks list in a config file, containing::

                c.NbConvertApp.notebooks = ["my_notebook.ipynb"]

            > jupyter nbconvert --config mycfg.py

To see all available configurables, use `--help-all`.
